# АВТО СИТИ ПРО Platform

Локальная демо-платформа автосалона: публичный сайт с каталогом, формами заявок, trade-in и сервисом, а также CRM-админка для обработки обращений.

## Требования

- Node.js 22 или новее
- npm

## Среда разработки и стек

- Редактор/IDE: Visual Studio Code
- Язык: TypeScript
- Разметка интерфейса: JSX/TSX-компоненты React, отдельные `.html` файлы в проекте не используются
- Фреймворк: Next.js App Router
- UI: React 19, Tailwind CSS, shadcn/ui-подход, Radix UI, Lucide React, Framer Motion
- Формы и валидация: React Hook Form, Zod
- Клиентское состояние админки: Zustand
- Локальное хранилище данных: SQLite через `node:sqlite`
- ORM/схема для дальнейшего развития: Prisma
- Сборка и проверки: npm, ESLint, TypeScript

## Быстрый запуск

1. Установите зависимости:

```bash
npm install
```

2. Создайте файл `.env` в корне проекта:

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/auto_city_pro?schema=public"
```

Для PostgreSQL сначала создайте базу `auto_city_pro`, затем выполните SQL-скрипт:

```bash
psql -U postgres -d auto_city_pro -f database/postgres-schema.sql
```

Код структуры БД находится в `database/postgres-schema.sql`, Prisma-схема PostgreSQL находится в `prisma/schema.prisma`.

3. Запустите локальный сервер:

```bash
npm run dev
```

4. Откройте проект в браузере:

```text
http://localhost:3000
```

Если порт `3000` занят, Next.js предложит другой порт, например `3001`.

## Как проверять заявки

Заявки сохраняются в локальную SQLite-базу `storage/auto-city-pro.sqlite`. При первом запуске база создается и заполняется стартовыми данными автоматически.

- Форма контакта попадает в админку: `/admin/leads`
- Покупка, кредит и тест-драйв из карточки автомобиля попадают в: `/admin/leads`
- Страхование попадает в: `/admin/leads` с источником `Страхование`
- Консультация попадает в: `/admin/leads` с источником `Консультация`
- Запись в сервис попадает в: `/admin/service`, а также создает лид в `/admin/leads`
- Trade-In попадает в: `/admin/trade-in`, а также создает лид в `/admin/leads`
- Фото из формы Trade-In сохраняются в `public/uploads/trade-in` и отображаются в заявке в админке
- После первой отправки формы ФИО и телефон сохраняются в браузере и автоматически подставляются в следующие формы на этом же устройстве
- Если клиент вошел или зарегистрировался через `/login`, его ФИО и телефон сразу подставляются во все формы услуг и заявок
- В карточке автомобиля гость не вводит ФИО и телефон повторно: кнопка заявки ведет на вход/регистрацию, а вошедший клиент отправляет заявку с данными из личного кабинета и своим комментарием
- Клиент после входа попадает в `/account` и не имеет доступа к CRM
- Сотрудник после входа попадает в рабочий раздел по роли: админ в обзор, менеджер в заявки, сервис-менеджер и механик в сервис, оценщик trade-in в trade-in
- Клиентская регистрация требует телефон, email остается необязательным; телефон и email клиента не могут совпадать с данными сотрудника
- Сотрудников можно добавлять только из админ-панели, публичной регистрации сотрудников нет
- В демо-версии роль существующего сотрудника нельзя сменить из таблицы команды без отдельной административной процедуры

Если админка уже открыта во время отправки заявки, нажмите кнопку `Обновить данные` в верхнем блоке админки.

## Функциональные требования АИС

Проект закрывает основную демонстрационную логику автоматизированной информационной системы автомобильного салона по модулям.

### Модуль "Учет автомобилей (Витрина)"

- Единый каталог автомобилей с маркой, моделью, годом, ценой, статусом, пробегом, типом двигателя, приводом и комплектацией
- Страницы брендов и отдельные карточки автомобилей
- Фотографии автомобилей в каталоге и на детальных страницах
- Управление наличием, видимостью, избранным статусом и ответственным менеджером в админке
- Фильтрация клиентской витрины по брендам и разделам каталога

### Модуль "Работа с клиентами (CRM)"

- Единая очередь лидов в `/admin/leads`
- Источники обращений: покупка, кредит, тест-драйв, сервис, trade-in, страхование, консультация и контактная форма
- ФИО, телефон, комментарий, интересующий автомобиль и история обращения фиксируются в заявке
- После первой заявки клиентские ФИО и телефон подставляются в следующие формы на этом устройстве
- В админке можно назначать ответственного и менять статус заявки

### Модуль "Продажи и финансы"

- Заявка на покупку автомобиля
- Заявка на кредит как отдельный сценарий CRM
- Trade-In с желаемым автомобилем, текущим автомобилем, годом, пробегом, VIN и комментариями
- Отдельная очередь trade-in в `/admin/trade-in`
- Возможность указать оценочную стоимость и статус решения по обмену

### Модуль "Склад, запчасти и автосервис"

- Запись клиента в сервис
- Отдельная сервисная очередь в `/admin/service`
- Назначение сервис-менеджера и механика
- Статусы сервисного заказа
- Базовая структура проекта готова для расширения под учет запчастей, остатков и списание материалов на ремонтные заказы

### Модуль "Отчетность и аналитика"

- Главный экран админки с KPI по заявкам, сервису, trade-in, каталогу и команде
- Отдельные KPI-блоки в модулях сервиса, trade-in и команды
- Данные хранятся в локальной SQLite-базе, что позволяет расширить проект до отчетов план/факт, конверсии и финансовой аналитики
